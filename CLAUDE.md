# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
pnpm test              # Run all tests (vitest)
pnpm test:watch        # Run tests in watch mode
pnpm typecheck         # TypeScript type checking without emitting
pnpm build             # Full build: generate icons → generate metadata → tsup build → fix ESM extensions
pnpm gen:icons         # Regenerate icon components from svg/ (run after changing SVGs)
pnpm gen:icon-metadata # Regenerate metadata.ts (run after adding/moving SVGs)
```

## Architecture

This is a Vue 3 icon component library (`@hmfw/icons`) — **681 Ant Design icons packaged as individual functional components**. The build system uses a code-generation pipeline followed by tsup compilation.

**Three-phase pipeline:**

1. **Code generation (`prebuild`)**: `scripts/generate-icons.ts` reads 681 SVGs from `svg/` and generates one Vue functional component per icon in `icons/`. `scripts/generate-icon-metadata.ts` generates `metadata.ts` with category/keyword data for search/filter utilities.
2. **Bundling (`tsup`)**: Two outputs — ESM (`bundle: false`, each icon as a separate file for tree-shaking) and IIFE/UMD (`bundle: true`, single file for CDN usage).
3. **Post-processing (`scripts/fix-extensions.js`)**: The ESM output has bare specifier imports (no `.js` extension) because tsup in `bundle: false` mode preserves source imports. This script rewrites relative `import`/`export` statements to append `.js`, and cleans up empty JS files from type-only modules.

**Key files:**
- `index.ts` — Public re-export entry: icon components, metadata, utility functions, types
- `types.ts` — `IconComponent` type (Vue `FunctionalComponent<SVGAttributes>`)
- `metadata.ts` — Auto-generated flat map of icon name → `{ keywords, category, tags? }`
- `utils.ts` — `searchIcons()`, `getIconsByCategory()`, `getAllCategories()`, `getAllIcons()`
- `icons/` — 682 auto-generated files: 681 named icon components + `index.ts` barrel export

**Icon naming convention:** SVG filenames use kebab-case (e.g., `bell-filled.svg`, `arrow-up.svg`). The generator converts these to PascalCase components: `-filled` suffix → `Filled` suffix, otherwise → `Outlined` suffix. Examples: `bell-filled.svg` → `BellFilled`, `arrow-up.svg` → `ArrowUpOutlined`, `home.svg` → `HomeOutlined`.

**Generated icon pattern:** Each icon is a `FunctionalComponent<SVGAttributes>` that returns a Vue `h('svg', {...})` call with `viewBox` from the source SVG, hardcoded `width: '1em'`/`height: '1em'`, `fill: 'currentColor'`, and `focusable: false`.

## Important Patterns

- **Never edit generated files manually.** Files under `icons/` and `metadata.ts` are regenerated by the prebuild scripts. Changes to SVGs belong in `svg/`; changes to icon rendering logic belong in `scripts/generate-icons.ts`; changes to categorization/keywords belong in `scripts/generate-icon-metadata.ts`.
- **ESM-only package.** `"type": "module"` in package.json. The IIFE build exists solely for CDN `<script>` usage.
- **`sideEffects: false`** is set — tree-shakers can safely eliminate unused icon imports.
- **Metadata is a static lookup table** — no runtime FS access. `getAllIcons()` iterates `Object.entries(iconMetadata)` and matches to actual exported components via the `toComponentName()` convention.
